# تطبيق القشرة المرِحة على ahmad-sarsor/tournament — بأمان كامل

## لماذا هذا آمن؟

- **البيانات لا تُلمَس**: كل البيانات في Firestore. التغيير CSS + سطر خطوط فقط — صفر تعديل على `assets/js/`، `config.js`، أو `firestore.rules`.
- **إضافة، لا استبدال**: `theme-fun.css` يُحمَّل *بعد* `styles.css` الأصلي ويبني فوقه. الشاشات غير المغطّاة (الإدارة، التوقّعات، الشجرة، النوافذ) تستمر بالعمل وتلتقط الألوان الجديدة تلقائياً لأن الأصل مبني على نفس المتغيّرات.
- **تراجع فوري**: حذف سطر `<link>` واحد (أو `git revert`) يعيد كل شيء.

## الخطوات

### 1) أنشئ فرعاً — لا تعمل على main مباشرة

```bash
git checkout -b design-refresh
```

### 2) أضف الملف

انسخ `theme-fun.css` (المرفق هنا) إلى `assets/css/theme-fun.css` في الريبو.

### 3) عدّل سطرين في `index.html` و`admin.html`

في `<head>` بعد سطر Google Fonts الموجود، أضف خط العرض:

```html
<link href="https://fonts.googleapis.com/css2?family=Baloo+Bhaijaan+2:wght@500;600;700;800&display=swap" rel="stylesheet" />
```

وبعد سطر `styles.css` مباشرة:

```html
<link rel="stylesheet" href="./assets/css/theme-fun.css" />
```

(اختياري) حدّث لون المتصفح: `<meta name="theme-color" content="#2f6bff" />`

### 4) جرّب محلياً قبل النشر

```bash
python3 -m http.server 8080
# افتح http://localhost:8080
```

`config.js` يعمل كما هو (مفاتيح Firebase عامة أصلاً) — ستتصفّح **البيانات الحقيقية قراءةً** وتتأكد من كل الشاشات: القائمة، البرنامج، الترتيب، صفحة مباراة، **admin.html** (سجّل دخول وتأكد من النماذج والأزرار)، والتوقّعات إن وُجدت.

### 5) انشر عبر Pull Request

```bash
git add assets/css/theme-fun.css index.html admin.html
git commit -m "قشرة مرِحة: theme-fun.css فوق التصميم الأصلي"
git push -u origin design-refresh
```

افتح PR وراجع، ثم ادمج في `main` — GitHub Pages يعيد النشر خلال ~دقيقة.

### 6) لو ظهر التصميم القديم عند بعض المستخدمين

كاش المتصفح فقط — تحديث قسري (Ctrl+Shift+R). ولضمانها للجميع أضف رقم نسخة للرابط:

```html
<link rel="stylesheet" href="./assets/css/theme-fun.css?v=1" />
```

### التراجع الكامل

```bash
git revert <رقم-الدمج>   # أو احذف سطر <link> من الملفين
```

## ما الذي سيتغيّر بصرياً؟

أزرق أكثر كهربائية في كل مكان، خط عناوين/نتائج مدوّر سمين (Baloo Bhaijaan 2)، نص Tajawal، تبويب مفعّل أزرق ممتلئ، زر أساسي «ينكبس»، بطاقات ترتفع وتميل عند التحويم، نتيجة المباراة داخل كبسولة (حمراء وقت البث)، لوحة نتيجة بلمعة، نقاط الترتيب زرقاء بخط العرض، وملمس انكباس على اللمس في الهواتف.

## ملاحظة الترقية الأعمق (اختيارية، لاحقاً)

هذا النظام هنا يضم أيضاً نسخة رموز كاملة (`tokens/`) ومكوّنات React — مفيدة إن قررتم يوماً إعادة بناء الواجهة. للتطبيق على الموقع الحالي، `theme-fun.css` هو الطريق الصحيح: أقل خطراً وأسرع.
